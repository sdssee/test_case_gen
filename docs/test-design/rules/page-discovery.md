# 连续页面深探规则

## 扫描和事务配合

扫描不是独立流程。进入页面时先从 DOM、可访问性树和实际页面状态做只读扫描；随后顺序操作；每次操作后局部扫描变化区域。如果出现未知按钮、条件字段、弹窗、抽屉或状态操作，将其加入当前事务或后续事务。结构发生大变化时再全量扫描。

## 必须真实操作

- 所有安全可操作元素必须实际点击、输入、选择或切换，不能只看到或只展开。
- 有限下拉选项逐项选择，记录选项值、操作前状态、页面变化、结果锚点和恢复结果。
- 分页按页面实际能力动态执行。存在什么控件就验证什么控件，不预设首页、末页、页码跳转或固定条数。
- 输入类记录正常、空值、格式、边界等页面实际可验证结果；动态选择记录搜索、清空、联动和无结果状态。
- 弹窗/抽屉验证打开、字段、校验、提交或取消，以及关闭后的页面状态。

## CRUD 与配置

- 创建必须成功，并能通过列表、详情或查询定位本轮测试对象。
- 编辑项逐个做有效变更，保存后重新打开验证持久化，并在可观察位置验证实际效果。
- 删除只作用于本轮带 `AI_TEST`、`CODEX_TEST` 或用户指定标识的数据，并验证对象不可再查询。
- 配置项采用单因素：默认/不配置为基线；每个可选配置单独设置。提交后依次验证提示、重开回显、真实效果和恢复。
- 条件配置字段在操作后动态发现并继续实探。无效或禁用分支记录拦截效果，不强行提交。

## 事实记录

原始观察追加到 `events.jsonl`，紧凑视图写入 `facts.json`，证据放在 `evidence/`。一个事务可以记录多个连续动作和多个选项，不为每个点击创建一条执行义务。事实应覆盖页面、角色、数据状态、元素能力、约束、精确选项、动作路径、真实结果、CRUD/配置闭环、测试对象生命周期、缺失/禁用能力和未决项。

证据引用必须位于当前 run-dir 的 `artifacts/discovery/evidence/` 且非空。截图只证明截图时刻；优先使用能定位动作和结果的文本、DOM 摘要、trace 或连续页面证据。

工具瞬时错误最多重试一次。真实失败直接记录；缺少权限或安全数据时记 pending 并继续其他项目；结束时只汇总一次缺口，不自动循环补跑。
