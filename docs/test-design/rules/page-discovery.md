# 连续页面深探规则

## 扫描与事务交替

进入页面时从 DOM、可访问性树和可见状态识别实际能力；随后执行一个相关功能事务。每次操作后局部重扫变化区域，出现新控件时加入当前或后续事务；结构明显变化和结束时才全量扫描。控件采用开放发现，不受预设清单限制。

扫描本身不是独立阶段，也不产生逐元素任务。内存中只维护本次尚未处理的新元素集合，完成后不持久化队列。

## 必须真实操作

- 所有安全可操作元素必须实际点击、输入、选择或切换，不能只看到或只展开。
- 有限下拉框逐项选择并记录选项值和可观察结果。
- 选择、导航、列表和容器类控件只验证页面实际发现的能力与分支；不存在的选项、按钮、状态或路径不臆造。
- 输入类至少验证正常值；必填输入验证空值；页面声明格式、最小值或最大值时分别验证无效格式、下边界和上边界，并记录 `input_class`。
- 一个结果需要按钮或其他控件触发时，检查动作必须写全输入、选择、触发和等待结果，并在 `used_element_refs` 中记录所有参与控件；不能把“只选择”写成已经产生提交结果。
- 弹窗、抽屉和详情验证打开、字段、提交或取消以及关闭后的页面状态。

## CRUD、编辑与配置

- 创建使用本轮安全测试数据并成功提交，随后查询或打开该对象验证实际创建结果。
- 普通编辑项逐项改变，保存后重新打开验证持久化回显和实际效果。
- 配置项采用单因素：默认/不配置作为基线，每个可选值分别设置、保存、重开、验证效果并恢复；暂不验证组合。
- 删除只作用于本轮测试对象，确认后查询不到该对象。
- 无效分支记录页面真实拦截，不强求保存成功。

## 完成与异常处理

页面最终扫描稳定、无未处理元素，输入分支和有限选项已在事务写入现场校验，CRUD/配置闭环或存在真实阻塞时，执行一次页面checkpoint并形成分组完成摘要。事务闭环在记录现场校验，计划阶段不再重复遍历全部深探规则。页面可验证问题必须自行操作；外部语义、权限、测试数据和环境问题统一写入 `open_items`，结束时只汇总一次。页面明确不存在的控件属于能力事实，不进入 `open_items`。

工具瞬时错误最多重试一次；真实业务失败记录一次，不自动循环。事实本身证明覆盖情况；截图、trace 和其他证据仅作可选诊断，不影响完成判断，也不供计划、用例、Review 或 Excel 使用。
