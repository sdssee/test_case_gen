## 生成规则

- 不得只写“异常值、边界值、性能测试需覆盖”等一句话；必须说明适用维度、具体场景、触发数据、操作路径、观察点、预期结果和恢复路径。
- 每个小功能块默认至少覆盖 DFT 功能、DFB 业务、DFS 安全或权限、DFR 可靠或数据一致、DFU 可用性；有接口、性能、部署、运维、兼容特征时追加对应维度。
- 页面元素类用例必须把 DFT 的正向流程、边界值、异常输入、逆向操作转化为可执行步骤，例如输入最大长度、空值、重复值、非法格式、取消/关闭/返回、重复提交和数据不变。
- 性能相关场景必须写入 `性能测试设计` Sheet，不得只放在功能测试用例备注中；性能场景应明确目标用户量/并发数、响应时间目标、数据量级、监控指标、通过标准和造数策略。
- 接口、超时、重试、幂等、数据一致、日志、告警、监控、回滚等系统性场景如果当前页面无法直接执行，应写入风险与待确认问题、性能测试设计或自动化建议，并说明需要的接口、日志、监控或环境支持。
- 破坏性、资源耗尽、并发极限等 DFX 极端场景不得在共享或生产样数据上直接执行；仅允许在隔离测试环境、压测环境或本次创建的测试数据上设计，并明确安全边界、回滚或清理策略。

## 用例落地格式

生成功能测试用例时，`测试数据`、`操作步骤`、`预期结果` 至少应体现：

1. 维度与场景：例如 `DFT-边界值`、`DFS-权限控制`、`DFR-幂等性`。
2. 输入或状态：正常值、边界值、非法值、重复值、无权限角色、异常状态、超时/重试条件、并发条件等。
3. 观察点：页面提示、字段校验位置、按钮禁用态、数据落库、列表刷新、状态流转、日志/审计、通知、接口错误码、监控指标。
4. 预期结果：成功、失败、拦截、数据不变、可恢复、可重试、回滚成功、告警触发或指标达标。

## 批次自检

每批完成前，应检查：

- 功能测试用例是否至少覆盖当前小功能块适用的 DFT、DFB、DFS、DFR、DFU 场景。
- 性能测试设计是否覆盖 DFP 响应时间、并发处理、大数据量和资源监控中适用场景。
- 风险与待确认问题是否记录当前无法验证但对产品质量重要的 DFI、DFD、DFO、DFX 场景。
- 页面元素覆盖清单中的输入、选择、按钮、弹窗、表格行操作、批量操作、分页、筛选、排序等元素是否都映射到可执行 DFX 场景或明确标注不适用原因。
