# 测试设计模板

本目录用于保存项目测试设计输出模板和字段说明。

- `codebuddy-test-design-template.xlsx`：测试设计 Excel 模板。
- `测试用例模板.xlsx`：测试系统导出的导入模板参考文件。
- `excel-template-spec.md`：模板字段说明。

使用 CodeBuddy 生成测试设计时，优先引用 `codebuddy-test-design-template.xlsx`。

如果输入包含需求文档、页面截图、原型图、页面 URL 或可访问系统，应先建立页面元素覆盖清单，再生成测试用例。页面中所有按钮、链接、菜单、图标按钮、筛选、分页、弹窗、表格行操作、上传、下载等可点击/可交互功能，必须关联功能测试用例；无法确认或不纳入测试的元素必须写明原因。

页面测试用例的业务逻辑应从设计文档、需求说明、接口文档和验收标准中抽取，再结合截图或实际页面补齐具体入口、控件和操作路径。文档只描述大概功能时，应使用页面入口补全场景，并把无法确认的业务规则登记为待确认问题。

`页面元素覆盖清单` 是覆盖追踪矩阵，不是测试用例 Sheet。所有功能测试用例必须写入 `功能测试用例` Sheet，所有性能测试场景必须写入 `性能测试设计` Sheet；覆盖清单只记录页面元素、业务依据、覆盖状态、发现方式、素材来源和关联用例 ID。

页面实探时必须保护已有数据：对已有数据可以查看、搜索、筛选、排序、分页、打开详情，或进入编辑/修改页面观察字段、校验和交互，但不能保存、提交或改变已有数据；新增、编辑、删除或状态变更只允许作用于本次实探自己创建且带有明显测试标识的数据。

功能测试用例应按模块、页面、业务流程和小功能块连续编排。同一小功能块的主流程、异常、边界、权限/状态、数据一致性等用例应放在附近，便于评审和执行。

如果用户提供测试数据，应优先用于页面实探和用例设计。正常数据用于确认成功路径，异常/边界数据用于观察页面真实校验、提示、跳转、状态变化和恢复路径；最终用例中的敏感数据必须脱敏。

`测试系统导入用例` Sheet 用于复制粘贴到测试系统导入模板。该 Sheet 字段顺序必须与 `测试用例模板.xlsx` 一致，内容从 `功能测试用例` Sheet 映射生成，不作为新的权威用例来源。

测试系统导入模板中的红色字段和必填字段必须按每条测试用例动态生成，不得固定写示例值或通用占位值。红色字段包括 `测试用例名称`、`测试步骤描述`、`测试步骤预期结果`、`测试类型`、`测试用例级别`、`执行方式`；必填字段至少包括 `测试用例名称`、`测试类型`、`测试用例级别`、`执行方式`。
