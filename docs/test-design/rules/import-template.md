# 测试系统导入规则

- 从 `docs/test-design/测试用例模板.xlsx` 创建独立副本，不修改原模板。
- 数据只来源于正式“功能测试用例”Sheet，不把性能或破坏性专项场景导入功能用例。
- 模块名称按 `scope.json.module_path` 逐级映射；系统编号、维护人、作者等无法可靠查询的字段保持空白，不编造。
- 用例序号从 1 连续生成，名称、步骤、预期、类型、级别、说明、前置条件和标签按正式用例逐行映射。
- 下拉字段只能使用模板允许值；执行方式默认“手工”，已有可运行自动化资产时才写“自动化”。
- 保留模板格式和数据验证；第 2 行起连续写入，不保留样例或中间空白行。
- 使用 `complete-deliverables` 与正式工作簿一起生成；单独生成时使用 `generate-import`。
