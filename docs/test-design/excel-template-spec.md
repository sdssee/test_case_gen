# 测试设计 Excel 模板规格

## Sheet 1：测试设计总览

字段：

- 项目/模块
- 需求名称
- 版本/迭代
- 测试负责人
- 需求来源
- 测试范围
- 不测范围
- 测试类型
- 测试环境
- 主要风险
- 准入条件
- 准出条件
- 待确认问题

## Sheet 2：需求用户故事拆解

字段：

- Story ID/需求 ID
- 用户故事/需求描述
- 角色
- 业务价值
- 验收标准
- 业务规则
- 前置条件
- 后置影响
- 依赖系统
- 待确认问题

## Sheet 3：测试场景矩阵

字段：

- 场景 ID
- Story ID/需求 ID
- 功能点
- 场景类型
- 正向/反向
- 测试维度
- 风险等级
- 优先级
- 是否生成用例
- 备注

## Sheet 4：功能测试用例

字段：

- 用例 ID
- Story ID/需求 ID
- 模块
- 功能点
- 用例标题
- 优先级
- 测试类型
- 前置条件
- 测试数据
- 操作步骤
- 预期结果
- 实际结果
- 执行状态
- 是否适合自动化
- 关联风险
- 备注

字段格式：

- `前置条件`、`操作步骤`、`预期结果` 必须使用 `1. 2. 3.` 换行分条。
- 示例：

```text
1. 测试环境已部署本次需求版本
2. 测试账号已完成注册且状态为启用
3. 用户已进入登录页
```

## Sheet 5：性能测试设计

字段：

- 性能场景 ID
- Story ID/需求 ID
- 业务链路
- 性能测试类型
- 目标用户量/并发数
- TPS/QPS 目标
- 响应时间目标
- 数据量级
- 测试时长
- 监控指标
- 通过标准
- 造数策略
- 风险说明
- 是否纳入本轮测试

## Sheet 6：风险与待确认问题

字段：

- 编号
- 类型
- 描述
- 影响范围
- 风险等级
- 建议处理方式
- 负责人
- 状态

## Sheet 7：自动化建议

字段：

- 用例 ID/场景 ID
- 自动化层级
- 自动化价值
- 自动化优先级
- 依赖数据
- Mock 需求
- 稳定性风险
- 建议框架/工具
- 备注

## 样式和可用性要求

- 所有 Sheet 表头冻结。
- 所有 Sheet 表头加粗并使用统一底色。
- `前置条件`、`操作步骤`、`预期结果`、`测试数据`、`备注` 等长文本列启用自动换行。
- 表格开启筛选。
- 枚举字段应使用固定值。
- 工作簿内保留示例行，便于用户理解填写方式。
