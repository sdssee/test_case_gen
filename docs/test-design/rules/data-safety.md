# 测试数据与操作边界

## 数据流通

- 本项目产物在用户允许的内网范围流通，不对内网IP、IPv6、环境URL、账号、密码、Token、密钥、Cookie、部署路径、内部服务地址或安全测试载荷执行识别、脱敏、占位符替换、警告或交付拦截；不执行敏感内容扫描。
- 用户提供的测试数据应优先用于页面实探和用例设计，并允许原样写入页面事实、JSON分片、正式Excel、导入Excel、批次状态和内部测试资产。
- DFX安全测试仍覆盖身份认证、权限控制、注入防护、会话、审计和产品自身的数据展示策略；框架不机械要求所有产品实现数据脱敏。

## 页面操作边界

- 只对本轮创建且带测试标识的数据执行新增、保存、编辑、删除、启停、审批、发布或状态变更。
- 对既有数据可以查看、搜索、筛选、排序、分页、打开详情、进入编辑页观察字段联动，以及打开危险操作确认弹窗；不得最终保存、提交、删除或改变状态。
- 无法判断数据归属时按既有数据处理。
- 修改共享环境、全局配置或影响后续Case的状态时，必须记录恢复方式并在验证后恢复。

## 测试标识与清理

- 本轮数据建议包含 `AI_TEST`、`CODEX_TEST`、日期或任务编号。
- 新增成功后继续验证详情、回显、配置效果和后续操作。
- 编辑验证保存、回显、生效及未修改字段保持不变。
- 删除分别验证取消和确认，并清理本轮创建的测试数据。

## 资产保护

- 框架升级不得覆盖或清空 `docs/test-assets/`、`docs/test-design/current/`、`docs/test-design/deliverables/`。
- 资产结构变化使用增量迁移，不使用空模板覆盖历史事实。
