## 使用原则

- 以可以连续实探和完整交付的最小功能范围切分，不机械按控件、义务或单条用例拆批。
- 每批复用同一 run-dir；事实只写入 `page-discovery.csv`，用例只写入既有 JSON 分片和正式工作簿，不在计划中复制正文。
- `batch-status.csv` 只记录断点、数量和下一步动作，不作为进入下一阶段的拒绝器。
- 发现页面可直接验证的待确认项时，回到当前连续实探会话增量执行，不重跑已经完成的分支。
- 所有批次完成后只进行一次最终语义 Review；局部问题就地修正，不启动自动返工循环。
