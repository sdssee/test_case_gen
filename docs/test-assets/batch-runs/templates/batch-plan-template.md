## 执行约束

- 范围超过一个二级菜单时，必须按批次逐个执行。
- 当前批次覆盖质量自检通过并更新 `batch-status.csv` 后，才能进入下一批；`batch-status.csv` 必须填写页面数、元素总数、已覆盖元素数、待确认元素数、功能用例数、性能场景数、异常用例数、边界用例数、权限/状态用例数和数据一致性用例数。
- 页面或功能点实探结果必须记录到 `page-discovery.csv`，并能追溯到页面元素覆盖清单和功能测试用例。
- 最终汇总只引用已归档批次成果和用例 ID，不得重新生成各批完整用例。
