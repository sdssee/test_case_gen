# 产品版图与历史资产规则

`product-map.xlsx`是可选的长期资产，不是本轮事实源，也不参与默认完成判断。本轮事实以run-dir的`page-discovery.csv`为准。

## 何时读取

- 用户指定依赖模块、历史用例或跨模块业务链路时，按需读取相关产品版图和归档。
- 当前功能与历史资产无关时不强制加载全量产品版图，避免增加上下文。
- 读取时跳过带`示例`或`DEMO`标记的模板行；示例行只说明字段和格式，不得作为产品事实、测试依据或Case输入。
- 不得仅凭对话记忆断言已有能力；引用历史用例时必须能定位到真实归档或用例ID。

## 何时询问用户

- 页面可直接验证的问题由模型继续点击、输入、选择或保存验证，不转给用户。
- 页面、需求、历史资产和现有数据都无法确认，且答案会实质改变范围、数据或预期时，才列为待确认问题。
- 用户确认后只增量补探受影响页面分支，并更新事实和Case规划；不重跑已实测分支。

## 按需同步

只有本轮明确启用产品版图时，才同步当前功能相关的模块、页面元素、业务对象、业务链路、用例索引、依赖和变更记录。同步不得保留模板示例行，也不得把`页面观察`或`DFX设计`冒充`已实测`。

正式测试设计归档到`docs/test-assets/modules/`，导入文件归档到`docs/test-assets/imports/`。跨模块用例保留一个主归属模块；依赖已有能力时可引用真实用例ID，不重复复制正文。

补充任务先读取现有交付件、相关事实分片和历史归档，识别缺口后只增量实探和更新受影响功能块。无需为补充任务创建自动返工队列或重新执行无关模块。
