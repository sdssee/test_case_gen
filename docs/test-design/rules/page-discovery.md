# 页面实探与元素覆盖规则

## 页面材料处理

涉及页面、截图、原型、页面 URL、菜单路径、浏览器或 computer use 时，必须先识别页面元素，再生成或补充用例。

材料优先级：

1. 需求、用户故事、验收标准和业务规则。
2. 用户提供的文档、截图、原型图、操作说明。
3. 可访问页面的实际 DOM、可见文案、菜单、按钮、链接、图标按钮、下拉项、开关、输入框、弹窗和表格行操作。
4. 浏览器或 computer use 观察到的页面行为。
5. 无法确认的信息登记为待确认问题。

## 深遍历要求

- 当前批次正式写测试用例前，如存在可访问页面、原型或桌面窗口，必须使用浏览器或 computer use 遍历当前批次所有可点击/可交互功能点。
- 具体写测试用例时，必须在对应页面或功能点内做深遍历，覆盖所有可点击、可输入、可测试元素。
- 页面所有按钮、链接、菜单、Tab、图标按钮、表格行操作、更多操作、批量操作、筛选、排序、分页、导入、导出、上传、下载、预览、弹窗、抽屉、确认框、开关、禁用态、空状态、错误态都必须记录和处理。
- 页面元素覆盖清单中的每个元素必须关联功能用例 ID、标记不适用、标记不测范围或登记待确认问题。

## 选择类控件

- 下拉框、级联选择、单选框、复选框、树选择、枚举筛选等选择类控件不得只展开查看选项。
- 必须选择代表性选项，观察后续字段、二级/三级下拉、列表数据、按钮禁用态、校验提示、默认值、清空重置、搜索结果和提交前提示变化。
- 对存在前后依赖的控件，必须记录每个关键选项触发的联动/依赖变化；无联动也要明确记录为无联动变化。
- 结果写入 `page-discovery.csv` 的 `选项取值/输入值` 和 `联动/依赖变化`，并反映到功能测试用例的操作步骤和预期结果。

## 输入类控件

- 输入框、搜索框、文本域、数字框、日期框、URL/地址、端口、邮箱、手机号、名称、编码等输入类控件不得只观察字段存在。
- 必须实际输入正常、异常、边界或用户提供的测试数据，触发查询、下一步、测试连接、预览、保存前校验等后续动作。
- 必须记录真实提示、字段校验位置、后续页面、结果分支/后续状态、禁用态和可恢复路径。
- 结果写入 `page-discovery.csv` 的 `选项取值/输入值`、`预期/观察行为`、`结果分支/后续状态`。

## 新增类流程

- 新增、创建、添加、新建、保存、提交、下一步、完成、测试连接等新增类流程必须实填实走。
- 在安全边界内实际填写测试数据并触发后续动作。
- 成功时必须进入详情页、下一级页面或后续配置页继续观察可点击/可输入/可测试功能。
- 失败时必须记录真实失败提示、停留页面、字段校验位置、可恢复路径和是否允许重新提交。

## 既有数据只读深探

- 对既有数据必须主动做只读深探：打开详情、进入编辑页观察字段和联动、尝试点击删除/停用/提交等危险按钮以打开确认弹窗或保存前校验。
- 不得点击最终确认、保存、提交或改变既有数据状态。
- 必须记录确认弹窗、二次确认、权限提示、取消路径、返回路径。
- 可以复制既有数据的非敏感字段，改名或改编码为带 `AI_TEST`、`CODEX_TEST`、当前日期或任务编号的新数据后新增，再用新数据探索后续页面。
