# 测试资产归档与模块能力索引规范

## 目标

测试用例和跨模块依赖不能依赖 AI 对话记忆。每次生成、评审或人工修改后的最终测试资产，必须回存到项目归档目录，并同步模块能力索引，供后续模块复用。

## 目录约定

```text
docs/test-design/
  outputs/   # 归档正式测试设计 Excel，保存评审后或人工修订后的最终版
  imports/   # 归档基于测试用例模板.xlsx 生成的测试系统导入文件副本
  indexes/   # 维护模块能力索引、跨模块依赖、可复用测试数据和归档清单
```

推荐归档路径：

```text
docs/test-design/outputs/<项目或需求编号>/<模块序号>_<模块名>_测试设计.xlsx
docs/test-design/imports/<项目或需求编号>/<模块序号>_<模块名>_测试系统导入.xlsx
```

## 模块能力索引

默认索引文件：

```text
docs/test-design/indexes/module-capability-index.xlsx
```

该文件至少包含：

- `已归档测试设计清单`
- `模块能力索引`
- `跨模块依赖关系`
- `可复用测试数据`
- `变更记录`

## 每次生成前

1. 读取 `docs/test-design/indexes/module-capability-index.xlsx`。
2. 如果用户指定依赖模块，优先读取该模块在索引中登记的归档测试设计。
3. 如果索引缺失但 `outputs/` 中存在相关归档文件，应读取归档文件并补充索引。
4. 不得仅凭 AI 对话记忆判断已有模块能力。

## 每次生成后

1. 将正式测试设计保存到 `outputs/`。
2. 如需导入测试系统，将导入文件副本保存到 `imports/`。
3. 将本模块提供的能力、关键状态、可复用前置条件、可复用测试数据、跨模块依赖和关联用例 ID 更新到 `module-capability-index.xlsx`。
4. 如果用户人工修改了测试设计，最终版必须回存归档，并更新索引中的变更记录。

## 跨模块用例管理

- 跨模块用例只保留一个主归属模块，避免重复维护。
- 被依赖模块通过 `跨模块依赖关系` Sheet、标签、备注和关联用例 ID 追踪。
- 当前模块依赖已有模块能力时，优先引用已有用例 ID 作为前置条件，不重复复制已有用例。
- 如果依赖关系本身存在风险，应新增跨模块链路用例，并登记到索引。
