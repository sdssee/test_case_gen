# 测试系统导入规则

- 从 `docs/test-design/测试用例模板.xlsx` 创建独立副本，不修改原模板。
- 数据直接来自 `function-cases.json`，与正式测试设计共享同一来源。
- 一条功能用例对应一条导入记录；性能或破坏性专项不伪装成功能用例。
- 模块名称按 `scope.module_path` 逐级映射；无法可靠查询的系统编号、维护人和作者保持空白。
- 名称、编号、配对步骤、预期、类型、级别、前置条件和标签按确定规则映射。
- 下拉字段只使用模板允许值；执行方式默认“手工”，已有可运行资产时才写“自动化”。
- 保留模板格式和数据验证；第2行起连续写入，不保留样例和中间空白行。
