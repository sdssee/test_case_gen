# 测试系统导入文件规则

## 生成方式

- 需要导入测试系统时，必须复制 `docs/test-design/测试用例模板.xlsx` 生成独立导入文件。
- 不得修改原模板文件。
- 不得在正式测试设计 Excel 中新增 `测试系统导入用例` Sheet。
- 导入内容必须从 `功能测试用例` Sheet 派生。
- 优先使用 `scripts/test_design_excel_tools.py generate-import` 或同等表头映射逻辑生成。
- 禁止批次脚本按固定列序号数组直接写入导入模板。

## 字段约束

导入文件字段必须严格匹配模板：

一级到五级模块系统编号/名称、其他模块系统编号/名称、测试用例系统编号、测试用例序号、测试用例名称、测试步骤描述、测试步骤预期结果、测试类型、测试用例级别、执行方式、测试用例说明、前置条件、维护人、标签、备注、作者。

- `测试用例名称` 来自 `功能测试用例.用例标题`，必须保留 `功能点-当前用例标题` 格式。
- `测试步骤描述` 来自 `功能测试用例.操作步骤`。
- `测试步骤预期结果` 来自 `功能测试用例.预期结果`。
- `前置条件` 来自 `功能测试用例.前置条件`。
- 测试系统自动生成字段不需要填写，不得编造值，例如 `测试用例系统编号`、`作者`、创建时间等。

## 下拉框与必填字段

- 红色字段和必填字段必须按每条测试用例动态生成，不得固定写示例值或通用占位值。
- 红色字段包括 `测试用例名称`、`测试步骤描述`、`测试步骤预期结果`、`测试类型`、`测试用例级别`、`执行方式`。
- 下拉字段只能填写模板允许值。
- `测试类型`：功能测试、性能规格测试、可靠性测试、兼容性测试、可维护性测试、安全性测试、易用性测试。
- `测试用例级别`：L1、L2、L3、L4。
- `执行方式`：自动化、手动。
- `执行方式` 默认填写 `手动`。只有存在已实现、可运行、可维护并覆盖该用例主要校验点的自动化资产，且本次交付明确按自动化导入或关联自动化资产时，才允许填写 `自动化`。
- 浏览器或 computer use 页面实探、AI 点击验证、自动化建议，都不能作为填写 `自动化` 的依据。

## 样式保留

导入文件必须保留模板中的 Excel 数据验证下拉框、必填字段样式、标红字段和自动生成字段空值。只能复制模板副本后填充内容，不能重建 Sheet、不能重画表头、不能改变边框、字体、填充、对齐、数字格式或下拉验证范围；新增数据行必须按模板第 2 行示例数据格式复制样式。生成后必须使用 `-ImportWorkbookPath <导入文件.xlsx>` 校验。
