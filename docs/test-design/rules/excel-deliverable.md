# Excel 交付规则

新运行固定在 `docs/test-design/current/<run-id>/`。每轮必须从同一个 `function-cases.json` 生成两个独立文件：

- `deliverables/正式测试设计.xlsx`：从正式测试设计模板复制生成，严格保留既有 8 个 Sheet 及顺序，并保留样式、冻结窗格、合并、校验、公式、下拉、隐藏设置、打印和筛选结构；长文本列应用固定的通用可读宽度，不把某次测试页面的内容长度写回模板。
- `deliverables/测试系统导入.xlsx`：从测试系统导入模板复制生成，不从正式 Excel 二次转抄。

交付回执必须返回 `delivery_dir`、`formal_workbook` 和 `import_workbook` 的完整路径。两个文件只能位于当前 run-dir 的 `deliverables/`，不向项目根目录或公共交付目录重复复制。

模型不直接编辑 Excel。确定性组装器只组合已固化资源：总览聚合真实范围、专项结论和设计完成条件；用户故事来自每功能 `design_context`；场景矩阵和功能用例的“功能点”统一使用Case级 `test_point`，步骤、预期和主验证必须来自同一关联检查；性能、风险、自动化分别来自计划专项结论和轻量自动化字段，交互型功能按功能链形成性能基线行，只有本地静态功能输出性能不适用原因；元素覆盖只把主控件、实际触发控件和明确 `branch_bindings` 计为覆盖，普通辅助使用仅写备注。风险编号和等级来自同一映射。组装器不发明业务结论或未提供资料。配对步骤分别编号并按功能连续写入。每次从干净模板生成，不保留样例、占位或中间空白行。

生成时保留并按实际数据区调整模板已有的表格、筛选和数据校验范围。正式用例和导入文件使用各自通用列宽画像，并按中英文显示宽度计算长文本行高。生成后单次技术校验：文件可打开、模板结构未损坏、数据行连续、无空测试点/空标题/内部ID/公式错误，两个文件的名称、数量、步骤和预期与结构化用例一致。失败保留明确错误，不自动重组、不返回页面实探或用例编写。
