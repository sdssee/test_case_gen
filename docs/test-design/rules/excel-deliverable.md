# Excel 交付规则

新运行固定在 `docs/test-design/current/<run-id>/`。每轮必须从同一个 `function-cases.json` 生成两个独立文件：

- `deliverables/正式测试设计.xlsx`：从正式测试设计模板复制生成，严格保留既有 8 个 Sheet 及顺序，并保留样式、冻结窗格、合并、校验、公式、下拉、隐藏设置、打印和筛选结构；长文本列应用固定的通用可读宽度，不把某次测试页面的内容长度写回模板。
- `deliverables/测试系统导入.xlsx`：从测试系统导入模板复制生成，不从正式 Excel 二次转抄。

交付回执必须返回 `delivery_dir`、`formal_workbook` 和 `import_workbook` 的完整路径。两个文件只能位于当前 run-dir 的 `deliverables/`，不向项目根目录或公共交付目录重复复制。

模型不直接编辑 Excel。确定性组装器只组合已固化资源：总览聚合真实范围、专项结论和设计完成条件，不再把功能验收标准拼成准出条件；用户故事来自每功能 `design_context`；场景矩阵和功能用例来自计划、`verification_focus` 与结构化用例；性能、风险、自动化分别来自计划专项结论和轻量自动化字段，性能不适用原因只写在原因列，存在实质稳定性风险的自动化结论写“有条件”；元素覆盖按主验证与辅助使用关系形成简短中文完整分支摘要并优先使用稳定结果标记，不输出“另有N项”、内部输入分类词、偶发样本或通用trigger占位语。风险编号和等级由同一映射写入场景矩阵、功能用例和风险Sheet。组装器不得自行补写业务目标、验收标准、性能或风险结论，也不得声称使用了未提供的需求资料。配对步骤分别编号写入操作和预期列，并按功能顺序连续写入有效 Case。每次从干净模板生成，不在旧交付件追加，不预建空行，不保留样例、占位内容或中间空白行。

生成时保留并按实际数据区调整模板已有的表格、筛选和数据校验范围，不得统一删除表格对象。正式用例和导入文件使用各自的通用列宽画像，并按中英文显示宽度计算全部长文本行高。生成后只做一次技术校验：文件可打开、模板结构未损坏、数据行连续、无空标题/内部ID/公式错误，两个文件的名称、数量、步骤和预期与结构化用例一致；场景矩阵的功能、主验证元素、优先级和DFX字段与功能用例一致；功能用例与自动化建议的适用性结论一致；页面元素不得以辅助使用替代主验证。Excel技术问题最多从干净模板重组一次，不返回页面实探或用例编写。
