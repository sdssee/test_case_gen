# 产品级测试资产库

`docs/test-assets/`保存业务项目运行后形成的内部历史资产，不作为默认客户交付件。本轮页面事实始终以run-dir的`page-discovery.csv`为准。通用框架仓库中的`product-map.xlsx`每个Sheet保留一行通用示例，用于说明字段和提供写入格式；读取和首次同步时必须忽略或清除示例行。

```text
docs/test-assets/
  product-map.xlsx  # 可选长期产品版图
  modules/          # 正式测试设计归档
  imports/          # 测试系统导入文件归档
  batch-runs/       # 运行断点、页面事实和JSON分片
  indexes/          # 兼容或补充索引
```

## 使用原则

- 当前功能需要历史用例、依赖模块或跨模块链路时，按需读取相关产品版图和归档；不强制加载全量资产。
- 外部测试目录只用于审计与改进通用规则；未获得用户明确授权时，不得把其中的产品名称、页面路径、地址、执行结果或本地运行记忆复制到本目录。
- 用户直接调用Skill，run-dir由内部工具创建或复用，不把初始化作为用户流程。
- `batch-status.csv`只记录进度、数量、路径和下一步，不作为质量门禁。
- `page-discovery.csv`记录`已实测/页面观察/DFX设计/待确认`，推断不得冒充实测。
- 正式测试设计归档到`modules/`，导入文件归档到`imports/`；产品版图仅在显式启用时同步。
- 外网升级保护整个`docs/test-assets/`，不得覆盖或删除内网真实资产。
