# 架构设计

## 原则

本项目采用轻量编排器、连续页面实探会话和自动事实编译器。执行状态来自阶段产物，不依赖模型记忆，也不为每个元素建立任务状态机。

## 组件

1. `session_runtime.py`：初始化、追加观察、事实编译、阶段边界校验、双向审查。
2. 页面工具：在同一会话中读取 DOM/可访问性树/可见状态并顺序操作。
3. `formal_assembler.py`：把 facts、plan、cases 映射成固定 8 Sheet，并生成独立导入工作簿。
4. `test_design_cli.py`：唯一命令入口。

## 开放发现与事务

扫描器只读。事务执行器顺序执行安全动作，操作后局部重扫。若页面结构变化，新的控件自然进入事实集合；架构不假设所有产品都有首页、末页、页码跳转或固定分页选项。

CRUD 与配置属于同一实探机制。配置变体只做单因素，分别记录默认、设置、提交、重开回显、实际效果和恢复，不进行组合爆炸。

## 写入隔离

阶段对上游只读，每个产物只有一个阶段可以写。审查执行 facts→plan→cases 和 cases→facts 双向追溯，并检查重复正文、格式、功能连续区块和交付映射。校验位于阶段边界，不参与页面每一步操作，因此不会形成重试风暴。
