## 测试类型解释

- 功能测试：验证系统实现是否符合需求规格，关注完整性和正确性。
- 性能规格测试：验证系统各项性能和规格指标。
- 可靠性测试：在各种负载或异常条件下验证系统长时间运行的稳定性和可靠性。
- 兼容性测试：验证系统是否兼容不同运行环境，包括协议适配、友商对接、客户端及浏览器适配、硬件适配、操作系统适配等。
- 可维护性测试：验证系统是否易于维护，能否快速测试、定位和修复故障。
- 安全性测试：验证系统面对各种安全威胁和攻击时的抵御能力，以及是否严格遵循安全规范和最佳实践。
- 易用性测试：验证界面及交互流程是否符合用户习惯，关注可理解性、便捷性、美观性、防呆设计、交互反馈等。

## 导入生成约束

- 下拉框字段只能填写模板或测试系统允许的选项。
- 自动生成字段必须保持空值。
- 必填字段必须随生成用例动态填充，不得保留模板示例值。
- 生成导入文件时必须复制 `docs/test-design/测试用例模板.xlsx`，只填充内容，不修改模板文件本身。
- 生成后的数据行格式、边框、字体、填充、对齐、数字格式和下拉验证范围必须沿用模板示例数据行。
