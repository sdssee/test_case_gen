# 页面实探与元素覆盖规则

## 动态能力扫描

- 从DOM、可访问性树、可见文案、悬停提示和实际页面状态识别菜单、Tab、按钮、图标、输入、选择、开关、表格、分页、弹窗、抽屉、CRUD、上传下载及页面跳转。
- 清单只是起点；发现规则中没有预设的新按钮或交互时，动态加入当前探索范围。
- 主要基于实际页面理解功能；需求、原型、截图和文档只作参考。

## 连续深探

- 在同一会话按业务事务连续执行，不把每个元素拆成独立任务或义务队列。
- 页面能验证的问题由模型实际点击、输入、选择、保存并观察，不询问用户代替实探。
- 只有页面、文档和现有数据都无法确认的内容才询问用户。
- 完成初次能力扫描后、执行受影响事务前，使用一次事实检查点集中列出需继续页面实探项和真正需用户确认项；不要在最终用例生成后才补问风险。
- 用户确认后，对变得可页面验证的内容只做增量补探，不重复`已实测`分支。

## 事实状态

`page-discovery.csv`使用`事实状态`字段：

- `已实测`：实际触发操作并观察到结果。
- `页面观察`：从页面、DOM或可访问性树确认，但未实际触发。
- `DFX设计`：基于功能和DFX设计，尚未在页面实测。
- `待确认`：当前页面、资料、权限或环境无法确认。

同一元素的多个分支状态相同时可以合并；状态不同才拆分必要记录。不得把部分选项实测后将全部选项标记为`已实测`。

## 选择与输入

- 每个有限选项都应实际选择并观察选中值、联动字段、按钮状态、提交或执行结果；每个选项规划独立baseline Case。
- 每个有效输入等价类分别实际输入并触发后续动作；必填空值、明确格式异常和边界由DFX扩展。
- 验证有限选项时，其他输入使用默认稳定值；验证输入等价类时，选择控件使用默认选项。选项Case和输入Case不得相互替代。
- 不默认对选项和输入做笛卡尔积；页面存在依赖或不同业务结果时才补充组合。

## CRUD与配置

- 新增、编辑和可选配置必须在本轮测试数据上实际保存并验证列表/详情回显和业务效果。
- 每个可选配置采用单因素遍历，覆盖不配置和分别配置；不默认组合爆炸。
- 编辑类表单的每个独立可修改字段分别执行一次保存、回显和生效验证；其他字段保持稳定值。
- 删除分别验证取消和确认，只删除本轮测试数据。
- 既有数据只读深探，可以观察详情、编辑字段、保存前校验和危险操作确认弹窗，但不得最终改变状态。

## 覆盖记录

- 页面元素覆盖清单中的每个元素关联用例ID、不适用、不测范围或待确认结论。
- `已实测`的选择、输入和CRUD记录必须包含实际数据、联动/观察行为和后续状态。
- `页面观察`与`DFX设计`可以形成Case，但步骤和预期不得表述为已经实测的页面结论。
- 关联用例ID时`是否已生成用例`必须为`是`；`覆盖状态=已覆盖`时必须存在有效关联用例ID。
- `适用DFX维度`必须包含关联用例实际使用的DFX维度；页面元素清单直接由本CSV编译，禁止另写一份不同步的数据。
- 页面能够安全验证的空格、长度、重复操作、错误反馈和状态变化不得继续保留为`待确认`。
- `关联用例ID`表示主验证归属，不记录仅用于导航、填表或触发保存的辅助使用。两个独立可配置字段不得关联同一个主验证Case。
- `已实测`不得仍标记为`待确认`；`不适用`、`不测范围`和`待确认`必须填写明确原因。
- 证据为可选诊断材料，不参与完成判断和门禁。

## 分页与页面生态

- 分页能力从实际页面动态识别，不预设一定存在首页、末页、跳页或固定每页条数。
- 每页条数下拉必须逐项选择并记录列表条数、总页数、当前页及按钮状态变化，每个选项规划独立baseline Case。
- 上一页、下一页、指定页跳转以及页面实际存在的首页、末页分别点击并观察数据集与页码变化。
- 单页数据只能证明翻页按钮禁用，不能替代每页条数、跳页或多页翻页能力。数据不足时复用一次性准备的本轮测试数据集连续验证，不为每个Case重复造数。
- 刷新、搜索区显示/隐藏、列设置及悬停后显示名称的图标均按实际能力分别登记和实探。
