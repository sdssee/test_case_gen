# 测试设计模板

本目录用于保存项目测试设计输出模板和字段说明。

- `codebuddy-test-design-template.xlsx`：正式测试设计 Excel 模板。
- `测试用例模板.xlsx`：测试系统导出的导入模板参考文件。
- `excel-template-spec.md`：模板字段说明。
- `archive-and-index-guidelines.md`：测试资产归档、模块能力索引和跨模块依赖维护规范。
- `indexes/module-capability-index.xlsx`：模块能力、跨模块依赖、可复用测试数据和归档清单索引。

使用 CodeBuddy 生成测试设计时，优先引用 `codebuddy-test-design-template.xlsx`。

如果输入包含需求文档、页面截图、原型图、页面 URL 或可访问系统，应先建立页面元素覆盖清单，再生成测试用例。页面中所有按钮、链接、菜单、图标按钮、筛选、分页、弹窗、表格行操作、上传、下载等可点击/可交互功能，必须关联功能测试用例；无法确认或不纳入测试的元素必须写明原因。

页面测试用例的业务逻辑应从设计文档、需求说明、接口文档和验收标准中抽取，再结合截图或实际页面补齐具体入口、控件和操作路径。文档只描述大概功能时，应使用页面入口补全场景，并把无法确认的业务规则登记为待确认问题。

`页面元素覆盖清单` 是覆盖追踪矩阵，不是测试用例 Sheet。所有功能测试用例必须写入 `功能测试用例` Sheet，所有性能测试场景必须写入 `性能测试设计` Sheet；覆盖清单只记录页面元素、业务依据、覆盖状态、发现方式、素材来源和关联用例 ID。

页面实探时必须保护已有数据：对已有数据可以查看、搜索、筛选、排序、分页、打开详情，或进入编辑/修改页面观察字段、校验、联动、多级子菜单和保存前提示，但不能保存、提交或改变已有数据；新增、编辑、删除、状态变更或其他敏感操作只允许作用于本次实探自己创建且带有明显测试标识的数据。

功能测试用例应按模块、页面、业务流程和小功能块连续编排。同一小功能块的主流程、异常、边界、权限/状态、数据一致性等用例应放在附近，便于评审和执行。

功能测试用例中的 `用例标题` 以及独立测试系统导入文件中的 `测试用例名称` 必须正式、简洁、可检索，避免口语化、聊天式或操作随笔式表达。由于测试系统导入后没有独立 `功能点` 字段，标题必须使用 `功能点-当前用例标题` 格式，例如 `搜索筛选-按智能体名称查询列表`。

如果用户提供测试数据，应优先用于页面实探和用例设计。正常数据用于确认成功路径，异常/边界数据用于观察页面真实校验、提示、跳转、状态变化和恢复路径；最终用例中的敏感数据必须脱敏。

生成或补充测试用例前，应读取 `indexes/module-capability-index.xlsx`。如果当前模块依赖其他模块，应读取索引中登记的依赖模块归档测试设计；不得仅凭 AI 对话记忆判断已有模块能力、已有用例、可复用测试数据或跨模块依赖。

正式测试设计工作簿不再新增 `测试系统导入用例` Sheet。需要导入测试系统时，必须复制 `测试用例模板.xlsx` 生成独立导入文件，并把 `功能测试用例` Sheet 中需要导入的用例映射填充到该副本；不得修改 `测试用例模板.xlsx` 原模板文件。

最终版测试设计应回存到 `outputs/`，测试系统导入文件副本应回存到 `imports/`，并同步更新 `indexes/module-capability-index.xlsx` 中的归档清单、模块能力索引、跨模块依赖、可复用测试数据和变更记录。

测试系统导入模板中的红色字段和必填字段必须按每条测试用例动态生成，不得固定写示例值或通用占位值。红色字段包括 `测试用例名称`、`测试步骤描述`、`测试步骤预期结果`、`测试类型`、`测试用例级别`、`执行方式`；必填字段至少包括 `测试用例名称`、`测试类型`、`测试用例级别`、`执行方式`。

测试系统导入模板中带下拉框的字段只能填写下拉允许值：`测试类型` 为功能测试、性能规格测试、可靠性测试、兼容性测试、可维护性测试、安全性测试、易用性测试；`测试用例级别` 为 L1、L2、L3、L4；`执行方式` 为自动化、手动。测试系统自动生成字段不需要填写，不得编造值。

`执行方式` 表示当前在测试系统中如何执行该条用例，默认填写 `手动`。只有已有可运行、可维护并覆盖该用例主要校验点的自动化脚本/API/流水线任务，且本次交付明确按自动化导入或关联自动化资产时，才填写 `自动化`；`自动化建议` 或“适合自动化”不能直接转成 `执行方式=自动化`。

独立导入文件必须保留 `测试类型`、`测试用例级别`、`执行方式` 的 Excel 数据验证下拉框，不能只写入合法文本值。
