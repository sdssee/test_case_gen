# 数据安全与脱敏规则

## 页面实探数据边界

- 使用浏览器或 computer use 实探页面时，只能新增、修改、删除或变更自己创建且带测试标识的数据。
- 测试标识建议包含 `AI_TEST`、`CODEX_TEST`、当前日期或任务编号。
- 对页面已有数据可以查看、搜索、筛选、排序、分页、打开详情，或进入编辑/修改页面观察字段、校验、联动、多级子菜单和保存前提示。
- 对已有数据不得保存、提交、确认修改、删除、审批、驳回、禁用、启用、发布、下线、分配、转移、批量处理或改变状态。
- 无法判断是否由自己创建的数据，一律按已有数据处理。

## 用户提供测试数据

- 如果用户提供测试数据，必须优先用于页面实探和用例设计。
- 正常数据用于校准成功路径预期。
- 异常、边界、非法或空值数据用于观察真实错误提示、禁用态、校验位置、接口/页面响应和可恢复路径。
- 如果用户提供的数据明确用于本次测试并带有测试标识，可以用于新增、保存、编辑、删除、状态变更等会落库的操作。
- 如果数据会修改已有数据或无法确认数据归属，只能用于查看、填入后观察校验、记录预期或设计用例，不得保存或提交。

## 脱敏要求

- 正式交付件、导入文件、批次账本、页面实探记录、临时脚本和产品版图不得写入疑似真实密钥、Token 或密码。
- 例如 `sk-...`、`secret=...`、`password=...`、`token=...`、`密钥:真实值` 都必须替换。
- 使用 `<valid_api_key>`、`<test_token>`、`<test_service_url>`、`<masked_phone>`、`<masked_account>` 等占位符。
- 生产账号、手机号、身份证号、真实地址、真实密钥、真实密码、真实 Token 不得原样输出到测试用例或产品版图。

## 升级保护

- 外网到内网普通框架升级不得覆盖、删除或清空 `docs/test-assets/`、`docs/test-design/current/`、`docs/test-design/deliverables/`。
- 标识：PROTECTED_ASSET_DIRS。
- `asset_schema_version` 或产品版图结构变化时，必须通过迁移脚本增量补齐，不能用空模板覆盖内网真实资产。
