# 连续页面深探规则

## 扫描与事务交替

进入页面时从 DOM、可访问性树和可见状态识别实际能力；发现输入元素后，模型结合页面语义、需求参考和功能推断声明各有效输入类，运行时立即返回由 DFX 策略按该元素实际属性生成的 `exploration_requirements`，随后据此执行相关功能事务。每次操作后局部重扫变化区域，出现新控件时同样先登记再交互；结构明显变化和结束时才全量扫描。控件采用开放发现，不受预设控件清单限制。

扫描本身不是独立阶段，也不产生逐元素任务。内存中只维护本次尚未处理的新元素集合，完成后不持久化队列。

## 必须真实操作

- 所有安全可操作元素必须实际点击、输入、选择或切换，不能只看到或只展开。
- 有限下拉框逐项选择并记录选项值、选中状态和可观察结果；默认选项存在其他安全选项时，每个选择控件只需在一个实际有效关联内先切换再切回，其他关联直接使用默认值。一次连续事务可以完成全部选项，但每个页面实际支持的有效关联保留独立检查点。输入类与选项共同改变提交效果时在同一检查的 `branch_bindings` 中记录双方；不枚举未实测或无意义的组合。
- 选择、导航、列表和容器类控件只验证页面实际发现的能力与分支；不存在的选项、按钮、状态或路径不臆造。
- 组合容器展开后递归扫描并登记可操作子控件；分页容器动态识别页容量、前后翻页、首末页、页码、跳页、刷新、列显隐等实际存在能力，出现什么验证什么，不用固定分页模板。
- 无可见文字的图标按钮依次读取可见文案、aria、title、tooltip；仍不明确时悬停并局部重扫。悬停只完成识别，安全点击并观察页面变化后才完成主验证；禁用、展示型或纯容器元素记录状态即可。
- 输入类验证页面实际支持的每个有效等价类并分别记录 `input_class`；必填输入验证空值；模型在登记时结合页面语义、需求参考和功能推断判断明确格式，页面或需求声明最小值、最大值时分别验证无效格式、下边界和上边界。输入类别不与选项机械组合；只有页面实际执行表明二者共同决定结果时，按已观察有效关联分别成Case。
- 一个结果需要按钮或其他控件触发时，检查动作必须写全输入、选择、触发和等待结果；`element_ref` 指向主交互控件，`used_element_refs` 记录全部参与控件，`branch_bindings` 只记录共同决定当前结果的分支。普通辅助使用不完成其他控件自己的独立分支，不能把“只选择”写成已经产生提交结果。
- 加载、禁用、完成、失败、超时和恢复等实际出现的状态直接记录在同一事务检查内；未出现的状态不臆造，也不为状态另建执行轮次或产物。
- 弹窗、抽屉和详情验证打开、字段、提交或取消以及关闭后的页面状态。

## CRUD、编辑与配置

- 创建使用本轮安全测试数据并成功提交，随后查询或打开该对象验证实际创建结果。
- 普通编辑项逐项改变，保存后重新打开验证持久化回显和实际效果。
- 配置项采用单因素：默认/不配置作为基线，每个可选值分别设置、保存、重开、验证效果并恢复；暂不验证组合。
- 删除只作用于本轮测试对象，确认后查询不到该对象。
- 无效分支记录页面真实拦截，不强求保存成功。

## 完成与异常处理

最后一个页面事务完成后执行一次全量扫描，明确记录稳定状态和空的未处理元素列表；若之后又产生新事务，原扫描自动失效，只需在全部操作结束时再扫一次。既定 `exploration_requirements` 已执行完成、CRUD/配置闭环或存在真实阻塞时，执行页面checkpoint并形成分组完成摘要。checkpoint只汇总仍未执行的既定分支，不临时推导新任务，也不否定已经记录的实探事实。事务结构和闭环在记录现场校验，计划阶段不再重复遍历全部深探规则。页面可验证问题必须自行操作；外部语义、权限、测试数据和环境问题统一写入 `open_items`，结束时只汇总一次。页面明确不存在的控件属于能力事实，不进入 `open_items`。

工具瞬时错误最多重试一次；真实业务失败记录一次，不自动循环。事实本身证明覆盖情况；截图、trace 和其他证据仅作可选诊断，不影响完成判断，也不供计划、用例、Review 或 Excel 使用。
